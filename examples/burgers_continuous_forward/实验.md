### 实验 A：基线 PINN（标准配置）

**目的**：建立性能基准，验证环境正确。

**步骤**：
- 确认 `config.yaml` 内容与你提供的一致（尤其 `N0: 50, N_b: 50, N_f: 10000`，`use_pde: true` 或未设置即默认为 true）。
- 运行训练：
  ```bash
  python train.py
  ```
- 训练完成后，checkpoint 保存在 `outputs/<日期时间>/checkpoints/last.ckpt`，记下路径，例如 `outputs/19-28-53/checkpoints/last.ckpt`。
- 使用 `3D.py` 加载 checkpoint 并计算相对 L2 误差：
  ```python
  # 在 3D.py 中修改 ckpt_path 指向此 checkpoint
  model = PINNModule.load_from_checkpoint(ckpt_path, ...)   # 使用你之前成功的加载方法
  plot_burgers_3d_interactive(model, 'data/burgers_shock.mat')
  ```
- 记录最终 L2 误差（预期 ~1.2e-3）。

**数据收集**：
- 训练日志（loss 曲线截图）可从 TensorBoard 获取。
- L2 误差数值。
- 保存生成的 `burgers_3d.html` 文件（可改名 `expA.html`）。

### 实验 D：纯物理驱动（移除初始条件数据）

**目的**：展示缺少初始条件“锚定”时解的偏移。

**策略**：直接从训练数据集中移除 `initial_condition`，只保留 PDE 残差和边界条件（边界条件仍给予部分约束，确保解唯一性）。

**步骤**：
- 复制 `config.yaml` 为 `configs/configD.yaml`。
- 修改 `configD.yaml` 中的 `train_datasets` 部分，删除 `initial_condition` 条目，**保留** `mesh_sampler` 和 `dirichlet_boundary_condition`。修改后如下：
  ```yaml
  train_datasets:
    - mesh_sampler:
        _target_: pinnstorch.data.MeshSampler
        _partial_: true
        num_sample: ${N_f}
        collection_points:
          - f
    - dirichlet_boundary_condition:
        _target_: pinnstorch.data.DirichletBoundaryCondition
        _partial_: true
        num_sample: ${N_b}
        solution:
          - u
  ```
  同时可设置 `N0: 0`（但 `N0` 不再被使用），保持 `N_b: 50, N_f: 10000`。
- 运行：
  ```bash
  python train.py --config configs/configD.yaml
  ```
- 评估 L2 误差，预期 **0.05~0.07**。

**数据收集**：
- L2 误差。
- 与实验 A 对比，观察解的整体相位偏移。

### 实验 E：观测噪声鲁棒性

**目的**：验证物理约束对观测噪声的滤波能力。

**策略**：修改 `read_data_fn` 以向参考解添加高斯噪声，这样所有使用该数据的初始/边界条件都会带上噪声。实现一个可切换的噪声读取函数。

**步骤**：
1. **修改 `train.py`**，在 `read_data_fn` 之后添加一个带噪声的版本：
   ```python
   def read_data_fn_noisy(root_path):
       data = pinnstorch.utils.load_data(root_path, "burgers_shock.mat")
       exact_u = np.real(data["usol"])
       # 添加 10% 高斯噪声
       noise_level = 0.1
       noise = noise_level * np.std(exact_u) * np.random.randn(*exact_u.shape)
       return {"u": exact_u + noise}
   ```
2. **在 `main` 函数中**，根据配置选择读取函数。修改 `main` 函数开头附近，增加一个条件选择 `read_data_fn`：
   ```python
   def main(cfg: DictConfig) -> Optional[float]:
       # ...（原有代码）
       # 根据配置选择 read_data_fn
       read_data_fn_noise = cfg.get("noise_level", 0.0) > 0.0
       if read_data_fn_noise:
           selected_read_fn = read_data_fn_noisy
       else:
           selected_read_fn = read_data_fn
       
       metric_dict, _ = pinnstorch.train(
           cfg, read_data_fn=selected_read_fn, pde_fn=selected_pde_fn, output_fn=None
       )
       # ...（后续代码不变）
   ```
3. **在 `configE.yaml` 中**，添加一个自定义参数 `noise_level: 0.1`，并保持其他配置同实验 A：
   ```yaml
   N0: 50
   N_b: 50
   N_f: 10000
   use_pde: true
   noise_level: 0.1   # 自定义噪声水平
   ```
4. 运行训练：
   ```bash
   python train.py --config configs/configE.yaml
   ```
5. 评估 L2 误差，预期 **< 0.01**（约 0.004 左右）。

**数据收集**：
- L2 误差。
- 与实验 C 在同等噪声下的纯数据模型对比（可额外运行一个 `configC_noisy.yaml`，即 `use_pde: false` 且使用 noisy read_fn，展示误差极大）。

### 实验执行检查表

| 实验 | 配置文件 | 关键配置差异 | 预期 L2 误差 |
|------|----------|--------------|--------------|
| A | `config.yaml` (默认) | 无 | ~1.2e-3 |
| B | `configB.yaml` | N0=10, N_b=10, N_f=500 | <2e-2 |
| C | `configC.yaml` | `use_pde: false` | >0.2 |

### 注意事项
- 对于实验 C 和实验 E 的纯数据版本，若需对比，可复用实验 C 的配置并指定 `noise_level`，但需同时设置 `use_pde: false`。这需要 `read_data_fn` 同样支持噪声。创建 `configC_noisy.yaml`：`use_pde: false, noise_level: 0.1` 即可。
- 所有实验均保持相同的随机种子 `seed: 1234`（已在配置中），确保可复现。
- 完成每组实验后，立即用 `3D.py` 计算 L2 误差并保存 HTML 图，避免后续 checkpoint 混淆。

以上方案完全基于现有代码架构，改动极小，能够高效完成全部对比实验。如有细节需要进一步澄清，随时提出。
