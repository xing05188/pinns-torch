## 五组对比实验详细实施方案（Discrete Forward Burgers）

以下方案围绕离散前向 Burgers 问题中的采样量、Runge-Kutta 阶数、网络容量与训练预算展开，全部基于现有 `train.py` 和 `configs/config.yaml`，**不需要修改 `pinn_module.py`**。

---

### 通用准备

1. **工作目录**：`examples/burgers_discrete_forward`
2. **基础配置**：`configs/config.yaml`
3. **参考数据**：`data/burgers_shock.mat`
4. **关注指标**：
   - 预测解 `u` 的误差
   - 离散时间推进的稳定性

---

### 实验 A：基线离散前向配置

**目的**：建立默认配置下的基准结果。

**配置**：
- 保持 `configs/config.yaml` 不变
- 默认关键参数：
  - `q: 500`
  - `N: 250`
  - `idx_t0: 10`
  - `idx_t1: 90`
  - 网络层结构 `layers: [1, 50, 50, 50, q+1]`

**运行**：
```bash
python train.py
```

**预期现象**：
- 作为后续所有离散前向实验的参照
- 对 `t_1` 时刻预测应达到较稳定的基线误差

---

### 实验 B：初始观测稀疏化

**目的**：考察初始时刻样本减少后，对离散时间推进质量的影响。

**配置**：
- 复制 `config.yaml` 为 `configs/configB.yaml`
- 将 `N` 从 `250` 降为 `50` 或 `100`
- 其余保持不变

**运行**：
```bash
python train.py --config-name configB
```

**预期现象**：
- 初始条件约束减弱
- `t_1` 的预测误差上升，曲线更容易出现局部偏差

---

### 实验 C：初始观测增强

**目的**：验证更多 `t_0` 观测点是否能进一步提升前向预测精度。

**配置**：
- 复制 `config.yaml` 为 `configs/configC.yaml`
- 将 `N` 提升到 `400` 或 `500`
- 其余保持不变

**运行**：
```bash
python train.py --config-name configC
```

**预期现象**：
- 初始时刻拟合更充分
- `t_1` 的预测误差通常低于基线或更加稳定

---

### 实验 D：Runge-Kutta 阶数消融

**目的**：检验较少阶段数对离散建模能力和最终误差的影响。

**配置**：
- 复制 `config.yaml` 为 `configs/configD.yaml`
- 将 `q` 从 `500` 改为较小值，例如 `100` 或 `200`
- 同步修改网络最后一层输出维度，使其仍为 `q + 1`
  - 例如 `q: 100` 时，最后一层为 `101`
- 其余参数保持不变

**运行**：
```bash
python train.py --config-name configD
```

**预期现象**：
- 时间离散分辨率下降
- 预测解在复杂区域更容易产生数值误差

---

### 实验 E：网络容量消融

**目的**：比较更小网络在离散前向问题上的表达能力。

**配置**：
- 复制 `config.yaml` 为 `configs/configE.yaml`
- 将网络改为更浅或更窄的结构，例如：
  ```yaml
  net:
    layers: [1, 20, 20, 20, q+1]
  ```
- `N`、`q` 与实验 A 保持一致

**运行**：
```bash
python train.py --config-name configE
```

**预期现象**：
- 拟合能力下降
- 离散推进误差在空间上更不均匀

---
