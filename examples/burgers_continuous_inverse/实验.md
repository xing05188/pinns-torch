## 五组对比实验详细实施方案（Continuous Inverse Burgers）

以下方案基于现有 `train.py`、`configs/config.yaml` 和当前训练封装设计，**不需要修改 `pinn_module.py`**。实验目标是同时考察反问题中的数据稀疏性、网络容量、参数初始化和训练预算对参数识别精度的影响。

---

### 通用准备

1. **工作目录**：`examples/burgers_continuous_inverse`
2. **基础配置**：`configs/config.yaml`
3. **参考数据**：`data/burgers_shock.mat`
4. **关注指标**：
   - 状态量 `u(t, x)` 的预测误差
   - 参数 `l1`、`l2` 的识别误差

---

### 实验 A：基线反问题配置

**目的**：建立反问题的性能基准，作为后续所有对比的参照。

**配置**：
- 保持 `configs/config.yaml` 不变。
- 默认设置为：
  - `N_u: 2000`
  - 网络层数 `layers: [2, 20, 20, 20, 20, 20, 20, 20, 20, 1]`
  - `extra_variables.l1: 0.0`
  - `extra_variables.l2: -6.0`

**运行**：
```bash
python train.py
```

**预期现象**：
- `u` 的误差较低。
- `l1`、`l2` 能收敛到真实参数附近，作为后续实验比较基准。

---

### 实验 B：观测点稀疏化

**目的**：验证观测数据减少后，反问题识别精度下降的幅度。

**配置**：
- 复制 `config.yaml` 为 `configs/configB.yaml`
- 将 `N_u` 从 `2000` 调整为 `500` 或 `800`
- 其余保持与实验 A 一致

**运行**：
```bash
python train.py --config-name configB
```

**预期现象**：
- `u` 的预测误差上升
- `l1`、`l2` 的识别曲线波动更明显，收敛速度变慢

---

### 实验 C：数据充足性对比

**目的**：测试更多观测数据是否能进一步提升参数识别稳定性。

**配置**：
- 复制 `config.yaml` 为 `configs/configC.yaml`
- 将 `N_u` 调整为 `3000` 或在可用数据范围内尽量提高到更大的值
- 其余保持与实验 A 一致

**运行**：
```bash
python train.py --config-name configC
```

**预期现象**：
- `u` 的拟合更平滑
- `l1`、`l2` 更接近基准值，方差更小

---

### 实验 D：网络容量消融

**目的**：比较较小网络在反问题中的表达能力不足会带来什么影响。

**配置**：
- 复制 `config.yaml` 为 `configs/configD.yaml`
- 将网络改为较窄结构，例如：
  ```yaml
  net:
    layers: [2, 10, 10, 10, 10, 1]
  ```
- `N_u`、`extra_variables` 与实验 A 保持一致

**运行**：
```bash
python train.py --config-name configD
```

**预期现象**：
- `u` 的拟合能力下降
- 反推出的 `l1`、`l2` 更容易偏离真实值

---

### 实验 E：参数初始化敏感性

**目的**：检验未知参数初值对优化过程和最终识别结果的影响。

**配置**：
- 复制 `config.yaml` 为 `configs/configE.yaml`
- 将初始参数改为更远离真实值的设置，例如：
  ```yaml
  model:
    extra_variables:
      l1: 1.0
      l2: -8.0
  ```
- `N_u`、网络结构与实验 A 保持一致

**运行**：
```bash
python train.py --config-name configE
```

**预期现象**：
- 初期损失下降更慢
- 若优化较稳定，最终仍应回到接近真实参数；否则会暴露反问题的初始化敏感性

---

### 实验执行检查表

| 实验 | 配置文件 | 关键差异 | 关注点 |
|------|----------|----------|--------|
| A | `config.yaml` | 默认配置 | 基线误差与参数识别结果 |
| B | `configB.yaml` | `N_u` 减少 | 稀疏观测下的退化程度 |
| C | `configC.yaml` | `N_u` 增加 | 数据充足时的稳定性提升 |
| D | `configD.yaml` | 小网络结构 | 容量不足导致的欠拟合 |
| E | `configE.yaml` | 改变 `l1/l2` 初值 | 参数初始化敏感性 |
