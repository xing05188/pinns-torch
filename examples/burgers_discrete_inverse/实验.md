## 五组对比实验详细实施方案（Discrete Inverse Burgers）

以下方案围绕离散反问题中的采样量、Runge-Kutta 阶数、网络容量和参数初始化展开，基于现有 `train.py` 与 `configs/config.yaml`，**不需要修改 `pinn_module.py`**。

---

### 通用准备

1. **工作目录**：`examples/burgers_discrete_inverse`
2. **基础配置**：`configs/config.yaml`
3. **参考数据**：`data/burgers_shock.mat`
4. **关注指标**：
   - 预测解 `u` 的误差
   - 参数 `l1`、`l2` 的识别误差

---

### 实验 A：基线离散反问题配置

**目的**：建立默认配置下的基准结果。

**配置**：
- 保持 `configs/config.yaml` 不变
- 默认关键参数：
  - `q: 81`
  - `N0: 199`
  - `N1: 201`
  - `idx_t0: 10`
  - `idx_t1: 90`
  - `extra_variables.l1: 0.0`
  - `extra_variables.l2: -6.0`

**运行**：
```bash
python train.py
```

**预期现象**：
- `u` 的预测误差作为基线
- `l1`、`l2` 能稳定逼近真实参数

---

### 实验 B：双时刻样本稀疏化

**目的**：检验观测点减少后，参数识别是否明显退化。

**配置**：
- 复制 `config.yaml` 为 `configs/configB.yaml`
- 将 `N0`、`N1` 同步降低，例如：
  - `N0: 50`
  - `N1: 50`
- 其余保持不变

**运行**：
```bash
python train.py --config-name configB
```

**预期现象**：
- 参数识别波动加大
- `u` 与 `l1/l2` 的收敛都比基线更慢

---

### 实验 C：样本分布不均衡

**目的**：测试两个时刻观测量不对称时，模型是否更依赖信息更充分的时刻。

**配置**：
- 复制 `config.yaml` 为 `configs/configC.yaml`
- 例如设置为：
  - `N0: 50`
  - `N1: 300`
- 或反过来设置 `N0` 更大、`N1` 更小，形成对照
- 其余保持不变

**运行**：
```bash
python train.py --config-name configC
```

**预期现象**：
- 参数识别对信息更丰富的时刻更敏感
- 可观察 `N0/N1` 不平衡是否导致优化路径偏移

---

### 实验 D：Runge-Kutta 阶数消融

**目的**：检验较少阶段数是否会降低离散反问题的表达能力。

**配置**：
- 复制 `config.yaml` 为 `configs/configD.yaml`
- 将 `q` 从 `81` 改为较小值，例如 `41` 或 `21`
- 同步修改网络最后一层输出维度，使其等于 `q`
- 其余保持不变

**运行**：
```bash
python train.py --config-name configD
```

**预期现象**：
- 离散时间表达能力下降
- 参数识别精度通常低于基线

---

### 实验 E：参数初始化敏感性

**目的**：检验未知参数初值是否会显著影响反问题收敛。

**配置**：
- 复制 `config.yaml` 为 `configs/configE.yaml`
- 将初始参数改为更远离真实值的设置，例如：
  ```yaml
  model:
    extra_variables:
      l1: 1.0
      l2: -8.0
  ```
- `N0`、`N1`、`q` 与实验 A 保持一致

**运行**：
```bash
python train.py --config-name configE
```

**预期现象**：
- 训练初期不稳定性更明显
- 若优化器足够稳健，最终仍应回到接近真实参数；否则会暴露初始化敏感性

---

### 实验执行检查表

| 实验 | 配置文件 | 关键差异 | 关注点 |
|------|----------|----------|--------|
| A | `config.yaml` | 默认配置 | 基线误差与参数识别 |
| B | `configB.yaml` | `N0/N1` 同时减少 | 稀疏观测下的退化程度 |
| C | `configC.yaml` | `N0/N1` 不均衡 | 信息分布不对称的影响 |
| D | `configD.yaml` | `q` 减少 | RK 阶数对识别的影响 |
| E | `configE.yaml` | 改变 `l1/l2` 初值 | 参数初始化敏感性 |

- 如果比较不同 `q`，务必同步更新网络输出层维度。
- 全部实验建议使用相同随机种子 `seed: 1234`。
