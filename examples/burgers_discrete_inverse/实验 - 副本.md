## 3. 对比实验设计

为保证实验量适中且结论清晰，采用 A~E 五组。

### A. 基线实验（默认配置）

- 目的：获得默认设置下的参考结果。
- 配置：`q=81, N0=199, N1=201, l1=0.0, l2=-6.0`
- 命令：

```bash
python examples/burgers_discrete_inverse/train.py
```

### B. 稀疏采样实验

- 目的：验证样本减少对识别性能的影响。
- 变更：`N0=50, N1=50`
- 命令：

```bash
python examples/burgers_discrete_inverse/train.py N0=50 N1=50
```

### C. 不均衡采样实验（正向）

- 目的：观察双时刻采样不均衡带来的偏置。
- 变更：`N0=50, N1=180`（可追加反向实验 `N0=180, N1=50`）
- 命令：

```bash
python examples/burgers_discrete_inverse/train.py N0=50 N1=180
```

### C2. 不均衡采样实验（反向）

- 目的：与 C 形成镜像对照，验证信息主导时刻是否反转。
- 变更：`N0=180, N1=50`
- 命令：

```bash
python examples/burgers_discrete_inverse/train.py N0=180 N1=50
```

### D. 参数初始化敏感性实验

- 目的：验证反问题对初值的敏感程度。
- 变更：`model.extra_variables.l1=1.0, model.extra_variables.l2=-8.0`
- 命令：

```bash
python examples/burgers_discrete_inverse/train.py model.extra_variables.l1=1.0 model.extra_variables.l2=-8.0
```

### E. RK 阶数消融实验

- 目的：验证 `q` 降低后模型表达能力变化。
- 变更：
  - `q=41`
  - `net.layers` 最后一层同步改为 `41`（必须与 `q` 一致）
- 推荐做法：复制 `config.yaml` 为 `configE.yaml` 后修改，再运行：

```bash
python examples/burgers_discrete_inverse/train.py --config-name configE
```

## 6. 最终结论模板（填空）

可直接用于汇报或作业正文：

1. 基线实验（A）在默认配置下稳定收敛，`u` 误差为 `0.0063108052`，参数识别结果可靠。  
2. 稀疏采样（B）使 `u` 误差升至 `0.1444412768`，性能显著退化，说明样本数量不足会明显损害反问题求解。  
3. 不均衡采样对照（C vs C2）显示：`N0=50,N1=180` 时 `u=0.0033085698`，而 `N0=180,N1=50` 时 `u=0.0924771428`，说明模型对 `t1` 时刻信息更敏感。  
4. 初始化敏感性（D）中，`l1` 收敛到 `1.07741094`，与其他组差异明显，表明参数初值会影响最终识别结果。  
5. RK 阶数消融（E）将 `q` 从 `81` 降到 `41` 后，参数量从约 `11.9K` 降到 `9.8K`，`u` 误差为 `0.0095526306`，较基线略差但仍保持可用，体现了精度与模型复杂度的折中。  

---

## 7. 注意事项

- 对比实验尽量一次只改一类因素，避免结论混淆。
- 所有实验保持同一随机种子，保证可比性。
- 若执行 E 组，务必检查 `q == net.layers[-1]`，否则会维度不匹配报错。
- 本例是无放回采样（`replace=False`），采样数不能超过该时刻可用点数；经验上 `N0<=199`、`N1<=201`。
